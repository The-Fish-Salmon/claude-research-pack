# Mode: `quick` (Desktop-lite)

A fast brief, 500–1,500 words. The user wants to get oriented on a topic, not read a
thesis chapter.

## Workflow (single context, sequential)

1. **Scoping** — abbreviated. One refined question + the top 2–3 things you'd need
   to verify to answer it. No formal methodology blueprint.
2. **Investigation** — 3–6 MCP calls total. Read at most 5 abstracts in detail;
   download at most 2 papers (only if they're load-bearing). For each downloaded
   paper, invoke `paper-capture`.
3. **Composition** — draft directly against the findings.
4. **Devil's Advocate** — single collapsed pass. Run the SKILL.md protocol but
   collapse the three checks if the deliverable is short. Verdict still required.
5. **Hand-off** — write `00_Inbox/research-{slug}-{date}.md` per a trimmed
   [report_lit_review.md](../templates/report_lit_review.md).

## Required sections

- Question & one-paragraph answer
- 3–6 key findings, each with a citation
- What we don't know yet (≤3 bullets)
- References (APA 7.0)
- Provenance (1–2 lines)

## Skip

- Bias auditing pass (light topic, not worth the cost).
- Ethics check unless the topic obviously triggers it.
- Editor pass (composer self-edits).

## Desktop-specific cautions

- Quick mode fits Desktop's no-parallelism constraint well — there's only one
  investigator-equivalent and one composer-equivalent in the Code variant too.
- Watch the temptation to "just briefly check one more angle" — quick mode means
  quick. If you find you need more, propose escalating to `lit-review` and ask the
  user.
