# Mode: `quick`

A fast brief, 500-1,500 words. The user wants to get oriented on a topic, not read a thesis chapter.

## Workflow

1. **Scoping** -- abbreviated. One refined question + the top 2-3 things you'd need to verify to answer it. No formal methodology blueprint.
2. **Investigation** -- 1x `investigator.md`. 3-6 MCP calls total. Read at most 5 abstracts in detail; download at most 2 papers (only if they're load-bearing).
3. **Composition** -- 1x `composer.md`. No separate synthesizer.
4. **Devil's Advocate** -- single pass after composition (collapsed checkpoint). Can block.
5. **Hand-off** -- write `00_Inbox/research-{slug}-{date}.md` per the [templates/report_lit_review.md](../templates/report_lit_review.md) but trimmed.

## Required sections

- Question & one-paragraph answer
- 3-6 key findings, each with a citation
- What we don't know yet (<=3 bullets)
- References (APA 7.0)

## Skip

- Bias auditor (light topic, not worth the cost).
- Ethics agent unless the topic obviously triggers it.
- Editor (composer self-edits).
