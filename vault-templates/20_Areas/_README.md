# 20_Areas

PARA-style **areas of responsibility** — ongoing concerns without a defined end (in contrast to projects, which finish).

Examples for a research vault:

- `20_Areas/teaching/` — TA materials, course notes
- `20_Areas/lab-ops/` — equipment SOPs, calibration logs
- `20_Areas/career/` — applications, talks, CV drafts
- `20_Areas/methods/` — reusable analysis code, simulation templates

## Difference from `10_Projects/`

| | Projects | Areas |
|---|---|---|
| Has an end date? | yes | no |
| Has a "done" state? | yes | no, just "current" |
| Archives when finished? | yes (`40_Archive/`) | no |

If the work has a deliverable and a deadline, it belongs in `10_Projects/`. If it's a standing concern you maintain indefinitely, it belongs here.

## Conventions

- One folder per area; flat structure inside is fine.
- Areas don't get auto-archived — you maintain them as long as they're relevant.
- `paper-capture` doesn't write here.
